🛡️ Vulnerabilities Found

🔓
Unencrypted HTTP
AmpliPi uses HTTP (not HTTPS) for its web interface. All data, including passwords, travels unencrypted and can be intercepted!
🔑
Default Credentials
Many users never change the default username/password. Attackers know these defaults and try them first!
🔌
Exposed API
The REST API has no authentication. Anyone on the network can control all zones and settings!
🖥️
SSH Enabled
SSH port 22 is open with password authentication. This allows remote access if the password is compromised!
📦
Outdated Software
Running old versions of Linux packages with known vulnerabilities that attackers can exploit!
📝
No Security Logs
No logging of failed login attempts or suspicious activity. Attacks go unnoticed!
